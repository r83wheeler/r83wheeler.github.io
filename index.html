Week 2

Feb 1 - 5, 2021

During the second week of the bootcamp I found myself falling behind. This was mostly due to my misunderstanding (or non at all) of the GitHub workflow. The problem I have is not being able to visualize what is happening behind the scenes. A major problem I encountered was resolving merge conflicts. It took me a while to realize what my problem was, and I made more problems for myself during that time. So, GitHub seems to be a challenge for me, but I am persistant. I won't give up until I am fully proficient with it.

What are my thoughts on pseudocoding? I can see the real benefits of pseudocoding. It's good to have an outline of a project before moving forward to tackle it. I spent a lot of years in construction, and we always had blueprints, estimates, ghantt charts, etc. It would be foolish to try and build a skyscraper without direction.

I'm intrigued by the responsive web technologies we are learning about. I can't help but wonder how it was initially created. I'm also excited to get into pseudocoding because I think it will really help me become a skilled programmer.

You can have more than you've got because you can become more than you are. Unless you change how you are, you'll always have what you've got.

Jim Rohn

I like that i'm able to reference external CSS and JavaScript files in my HTML. I think there are a lot of advantages to seperating HTML and code. For one, it makes HTML, CSS and JavaScript easier to read and maintain.

What are some objects that my blog incorporates? Well, in JavaScript, almost everything is an object. I just added a scroll to top button to my blog. The scrollFunction is an example of an object.

My thoughts on pseudo coding haven't changed. I still find it very usefull, although I am still struggling on the Codewars Katas.
