I'm intrigued by the responsive web technologies we are learning about. I can't help but wonder how it was initially created. I'm also excited to get into pseudocoding because I think it will really help me become a skilled programmer.

Week 4

Feb 15 - 19, 2021

What was one thing that I liked and disliked about each of the new topics covered in prework?

Git and GitHub - What I like about Git and GitHub is that I'm able to save versions of my projects and know that whenever I come back to them, they will be there, unaltered. I had a lot of trouble using git and github initially. One small misstep left me lost and I ended up creating a lot more problems for myself while trying to fix the original problem.

Chrome Dev Tools - There isn't anything that I particularly disliked about Chrome Dev Tools. What I do like is the ability to test JavaScript code in the console. It's been very helpful for me to tryout for loops, for-in loops, for-of loops, while, and do-while loops. The more I can practise these, the better I will become as a programmer.

Visual Studio Code - There isn't anyting that I don't like about VS Code. As a matter of fact, it's my preferred text editor. One feature I really enjoy is the color coded text that is used to differentiate the syntax. The formatting also makes the code a lot easier to follow.

HTML - Again, nothing here to dislike. One thing I really like, is the ability to us hyperlinks. Hyperlinks allow us to link documents to other documents or resources, link to specific parts of documents, or make apps available at a web address.

CSS - One important thing I learned about CSS is type selectors. These select an HTML tag/element and add a CSS rule to them. I find it to be pretty amazing that I can us a span element, wrap it around some text, then assign that element to place a yellow background color to be applied to it. There are so many options that really piques my curiousity about CSS.

Pseudo Code - We learned about Pseudo Code, which I can see is going to be very important. Asking the questions, what, how, why, when and sometimes who, will help us as programmers to understand the problem at hand and bring us to a solution.

I'd say for me personally, what hindered my progress these last four weeks is being in a remote working environment. I find it to be easier to learn if i'm there in person around my peers and instructors. Sometimes it can be hard for me to articulate the problem i'm experiencing. Having the ability to share my screen on zoom has diminished this issue, but there still are a lot of pros to in person learning that I don't get in a remote environment.

I think my persistance really helped me overcome challenges. Searching through heaps of information until I found the solution can take a lot of persistance. Especially, if you're not exactly sure where to begin looking. Which is where I found myself most of the time. I think taking good notes, and really trying to understand what is being asked of me really helps to overcome challenges.
